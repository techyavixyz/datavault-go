Emergency Access & Seeding
Environment variables you can set before starting the container to recover access or seed an initial admin account.
Variable Purpose Condition
ADMIN_USERNAME Username for the seeded super admin account Only applied when no users exist yet
ADMIN_PASSWORD Password for the seeded super admin (used with ADMIN_USERNAME) Only applied when no users exist yet
RESET_ADMIN_PASSWORD Resets the first super admin's password to this value on every startup ⚠ Remove immediately after use
RESET_ADMIN_PASSWORD is applied on every container start while the var is set. Remove it from your docker-compose.yml or environment immediately after recovering access.
